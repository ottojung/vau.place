Static types vs contracts

Everybody these days seems to like static type systems. But there is an obvious advantage of dynamic languages - you can express more at runtime. Contracts check types at runtime and thus allow for stuff like: 
(define/contract bitcoin-price
  (if (system "ping -c 1 rate.sx") number? false?) ; `bitcoin-price` is a number iff we can query rate.sx
  (string->number (shell "curl rate.sx/10BTC")))   ; `string->number` returns false on error
