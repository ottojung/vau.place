Static types vs contracts

Everybody these days seems to like static type systems. Dynamically typed languages started to adopt some forms of gradual typing. But there is an obvious advantage of dynamic languages - you can express more at runtime. Contracts check types at runtime and thus allow for stuff like: 
(define/contract bitcoin-price
  (if (system "ping -c 1 rate.sx") number? false?) ; check if there is a connection
  (string->number (shell "curl rate.sx/10BTC")))   ; convert to number right away
